git commit -m "풀빌라 페이지 완성, 자잘한 디테일 전 백업"
git push


# 깃 기본 업데이트 작업 순서
git add .  이거 쓰면 새로 만든 것도 올리기
1. 변경된 파일 모두 스테이지에 올리기  
git add -A

2. 변경 내용 커밋하기 (메시지는 상황에 맞게 적기)  
git commit -m "변경 내용 간단히 적기"

3. 원격 저장소(깃허브)에 반영하기  
git push

[복사 붙여넣기 단축키]  
- 복사: Ctrl + C  
- 붙여넣기: Ctrl + V  
- 터미널 붙여넣기: 
Shift + Insert

[깃 내려받기(다운로드) 방법]

1. 원격 저장소에서 최신 내용 가져오기  
git pull
2. 새 프로젝트 처음 내려받기 (클론)  
git clone https://github.com/sadcorea/escort-girl.git

[파일 수정 여부 확인 방법]
git status
   - 수정된 파일, 새로 추가된 파일, 삭제된 파일 모두 보여줘요.  
   - 예) "modified: 파일명" (수정됨), "deleted: 파일명" (삭제됨)

git diff
   - 수정된 부분을 자세히 보여줘요.  

요약:  
- git status로 뭐가 바뀌었는지 먼저 확인  
- git diff로 구체적인 수정 내용 보기  



# 참고  
- 변경된 파일(수정, 추가, 삭제) 모두 자동으로 인식함  
- 커밋 메시지는 나중에 무슨 변경인지 알 수 있게 적기  
- 만약 푸시 실패하면 ‘git pull’로 최신 상태 받고 다시 시도
